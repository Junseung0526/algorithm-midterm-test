# 탐욕 알고리즘 기초 (Greedy Algorithm Basic)

## 1. 개요
매 순간 최적이라고 생각되는 선택을 하여 전체 최적해에 도달하려는 설계 기법입니다. 지금 당장의 이득만을 고려하는 '근시안적' 선택이 특징입니다.

## 2. 주요 알고리즘 분석

### 2.1. 분할 가능 배낭 문제 (Fractional Knapsack)
- **방식**: 물건을 쪼개서 넣을 수 있다고 가정하고, 단위 무게당 가치(가성비)가 높은 순서대로 배낭에 채우는 방식.
- **시간 복잡도**: $O(n \log n)$ (가성비 기준 정렬 시간이 지배적)
- **시험 포인트**: 
    - 0/1 배낭 문제(물건을 쪼갤 수 없음)와 달리 탐욕 알고리즘으로 **최적해**를 보장함.
    - 정렬 후 그리디하게 선택하는 전형적인 유형.

### 2.2. 작업 스케줄링 (Job Scheduling)
- **방식**: 이익이 큰 작업부터 선택하되, 다른 작업의 기회를 방해하지 않도록 마감일 직전에 배치하는 방식.
- **시간 복잡도**: $O(n^2)$ (최악의 경우 빈 슬롯을 찾는 과정 포함) 또는 정렬 포함 $O(n \log n)$.
- **시험 포인트**: 
    - 단순히 이익만 높은 순으로 고르는 것이 아니라, **시간 배치 전략**이 함께 고려되어야 함.
    - 그리디하게 이익을 챙기면서도 자원(시간)을 효율적으로 사용하는 예시.

## 3. 탐욕 알고리즘 성립 조건 및 비교

| 항목 | 내용 | 비고 |
| :--- | :--- | :--- |
| **탐욕 선택 속성** | 현재의 선택이 나중의 선택에 영향을 주지 않음 | 핵심 조건 1 |
| **최적 부분 구조** | 부분 문제의 최적해가 전체의 최적해가 됨 | 핵심 조건 2 |
| **분할 가능 배낭** | 탐욕 알고리즘으로 최적해 보장 | 가성비 정렬 |
| **0/1 배낭** | 탐욕 알고리즘으로 최적해 보장 불가 | DP로 해결 |
