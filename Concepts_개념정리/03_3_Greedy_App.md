# 탐욕 알고리즘의 응용 (Greedy Algorithm Applications)

## 1. 개요
탐욕 알고리즘은 최적해를 보장하는 데이터 압축(허프만 코딩)부터, 최적해에 가까운 답을 빠르게 찾아야 하는 복잡한 문제(집합 커버)까지 폭넓게 응용됩니다.

## 2. 주요 알고리즘 분석

### 2.1. 허프만 코딩 (Huffman Coding)
- **방식**: 문자의 빈도수에 따라 가변 길이 부호를 부여함. 빈도가 낮은 두 노드를 반복적으로 합쳐 트리를 구성하는 방식.
- **시간 복잡도**: $O(n \log n)$ (우선순위 큐 사용 시)
- **시험 포인트**: 
    - **접두어 부호(Prefix Code)** 성질을 만족하여 해독 시 모호함이 없음.
    - 데이터 압축 효율을 극대화하는 **최적 정적 이진 코드**임.

### 2.2. 집합 커버 문제 (Set Cover Problem)
- **방식**: 전체 원소를 커버하기 위해, 매 단계에서 아직 커버되지 않은 원소를 가장 많이 포함하는 집합을 선택함.
- **시간 복잡도**: $O(n \cdot m)$ (집합 수 $n$, 원소 수 $m$)
- **시험 포인트**: 
    - **NP-완전 문제**로 최적해를 찾기 매우 어려워 탐욕법으로 **근사해**를 구함.
    - 현실 세계의 자원 할당, 기지국 배치 문제 등에 활용됨.

---

## 3. 응용 알고리즘 비교 (요약)

| 알고리즘 | 목적 | 선택 전략 | 해의 성질 | 비고 |
| :--- | :--- | :--- | :--- | :--- |
| **허프만 코딩** | 데이터 압축 | 최소 빈도 노드 결합 | **최적해** | 접두어 코드 |
| **집합 커버** | 자원 최적화 | 최대 미커버 원소 포함 집합 | **근사해** | NP-완전 |

---
**Tip**: 탐욕 알고리즘이 항상 최적해를 보장하지는 않지만, 구현이 간단하고 속도가 빨라 실무적으로 매우 중요합니다!
