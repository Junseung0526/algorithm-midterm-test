# 03-3. 탐욕 알고리즘의 응용 (Huffman & Set Cover) - 중간고사 필승 요약

## 💡 Why? (왜 배우는가?)
- **효율적 압축**: 자주 나오는 데이터는 짧게, 드물게 나오는 데이터는 길게 표현하여 저장 공간을 아끼기 위함입니다.
- **최적의 선택**: 수많은 선택지 중에서 가장 가성비 좋게 전체를 아우르는(Covering) 전략을 세우기 위함입니다.

---

## 🚀 핵심 알고리즘 1줄 암기 포인트

1. **허프만 코딩 (Huffman)**: "빈도수 낮은 놈들끼리 묶어서 트리 만들기! 많이 나오면 짧은 코드 부여."
2. **접두어 부호 (Prefix Code)**: "어떤 코드도 다른 코드의 앞부분이 아니야! 덕분에 끊어 읽기(해독)가 가능."
3. **집합 커버 (Set Cover)**: "아직 안 된 놈들을 제일 많이 포함하는 집합부터 고르기! 정답(최적해)은 아닐 수 있지만 꽤 괜찮은 정답(근사해)을 줌."

---

## 📊 허프만 트리 구성 단계 (시험 단골 계산 문제)

1. 각 문자의 빈도수를 센다.
2. 빈도수가 가장 낮은 **두 노드**를 선택해 합친다.
3. 합친 노드의 빈도수는 두 자식의 합이 된다.
4. 노드가 하나 남을 때까지 반복한다.
5. 왼쪽 간선에는 0, 오른쪽 간선에는 1을 부여한다.

---

## ❓ 시험 빈출 Q&A

**Q1. 허프만 코딩이 '가변 길이 부호'임에도 불구하고 해독 시 모호함이 없는 이유는?**
- **A**: **접두어 성질(Prefix Property)**을 만족하기 때문입니다. 어떤 문자의 부호도 다른 문자 부호의 시작 부분(접두어)이 되지 않도록 설계되었습니다.

**Q2. 집합 커버 문제에서 그리디 알고리즘을 사용하는 이유는?**
- **A**: 이 문제가 **NP-완전(NP-Complete)** 문제이기 때문입니다. 모든 조합을 다 따져서 최적해를 찾으려면 시간이 너무 오래 걸려, 탐욕법으로 빠르게 근사해를 찾습니다.

**Q3. 텍스트 "AAAAAABCCD"를 허프만 코딩했을 때, 'A'에 부여될 코드의 길이는?**
- **A**: 빈도수가 가장 높으므로 가장 짧은 코드(보통 1비트)가 부여될 가능성이 큽니다. (실제 트리를 그려서 확인 필요)
